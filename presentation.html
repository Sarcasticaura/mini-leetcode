MiniLeet Coding Platform

A Server-Less, Client-Side Algorithm judge Environment

Subject: Software Technology Version: 1.0.0 Author: Developer

Introduction & Problem Statement

Project Background

  • The Challenge: Traditional coding platforms (LeetCode, HackerRank) require expensive remote compiler servers, complex Docker sandboxes, and stable high-speed APIs.
  • Educational Need: CS students require a lightweight, zero-latency, local-first playground to instantly verify code correctness.
  • The Solution: Mini LeetCode executes, logs, and asserts code completely in the user's browser, providing a zero-cost, portable evaluation environment.

Zero Latency

Compiler runs locally in the browser, providing instant evaluation feedback.

Serverless Architecture

Reduces setup complexity and hosting costs to zero.

System Architecture

High-Level Design

  • Single Page Interface (SPA): Vanilla HTML5 and CSS3 views toggled directly in the DOM. No React/Vite overhead, allowing raw local file execution.
  • Monaco Editor: Integrated via require.js AMD loader, providing professional autocompletes, themes, and code diagnostics.
  • CodeRunner Abstraction: Coordinates user configurations, test suites inputs, and execution parameters, and routes them to isolated threads.

HTML5 Web Workers

Runs execution processes in background threads, isolating the main UI from freezing.

Local State Persistence

Synchronizes drafts, user streaking status, submissions logs, and custom problems inside localStorage.

In-Browser Compilation Sandbox

JavaScript Worker Engine

  • Security Sandbox: Code executes in a separate thread. Workers are prohibited from accessing the page DOM or cookie values.
  • Infinite Loop Protection: Main thread implements a watchdog timer. Submissions are terminated if execution exceeds 3 seconds.
  • Console Capture: Intercepts `console.log()` to capture printing streams directly in the output panel.
// Inline Web Worker instantiation
const blob = new Blob([jsWorkerCode], 
  { type: 'application/javascript' }
);
const worker = new Worker(
  URL.createObjectURL(blob)
);
worker.postMessage({ code, args });

WebAssembly Python Engine

Pyodide Web Worker Integration

  • WASM Execution: Loads **Pyodide** (CPython compiled to WebAssembly) asynchronously via CDN inside a persistent worker.
  • Parameter Serialization: Encodes JS arrays/strings/booleans into JSON, parses them in Python using `json.loads()`, and converts results back to JSON.
  • Performance Optimization: Reuses the Pyodide worker across code runs to prevent loading latencies, but kills and recreates the worker if a timeout (8 seconds) triggers.

UI/UX Design & Styling Aesthetics

Visual Design System

  • Glassmorphic Cards: Panels styled with semi-transparent dark backgrounds, blurred backdrops, and fine borders.
  • Responsive Views: Fluid grid columns that automatically adapt the code editor and description split panes to standard displays.
  • Micro-Animations: Glowing borders on correct submissions, pulse states on streak flames, and active scaling transitions.

harmonious Color Theme

Dynamic Problem Creator Form

Expanding the platform

  • Full Creator Panel: Input title, difficulty, category, descriptions (HTML formatted), parameter arrays, and default starters.
  • JSON Parser: Automatically parses arguments lists and return definitions, validating code entries prior to insertion.
  • Local Syncing: Appends custom challenges into the master problems set. They load, compile, run, and calculate metrics in local storage.

Simple Integration

Persistence

User Analytics & Streaking Dashboard

Engagement Metrics

  • SVG Circular Progress Meter: Dynamically calculates and visualizes completion metrics.
  • Detailed Submission History Logs: Lists dates, execution states, runtimes, and languages. Highlights a **"Restore Code"** button to load past drafts back into Monaco.

Submissions Tracking

Conclusion

Project Summary

  • Offline-First Coding: A highly responsive development sandbox that operates entirely in the browser.
  • Robust Security & Timing: Complete client-side isolation using Web Workers and watchdog timers to prevent infinite loop tab freezing.
  • Highly Portable: Packaged as a single ZIP file containing the code, IEEE SRS PDF document, and interactive presentation.
